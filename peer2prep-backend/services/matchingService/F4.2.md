# F4.2 - Matching Service to Collaboration Service Integration

## Overview

This document describes the HTTP integration between the Matching Service and Collaboration Service. When a match is found, the Matching Service calls the Collaboration Service via HTTP to create a collaboration session, then returns the `collaborationId` to both users for redirection.

## Architecture

```
┌─────────────────┐         ┌─────────────────┐
│  User A Browser │         │  User B Browser │
└────────┬────────┘         └────────┬────────┘
         │                           │
         │ WebSocket                 │ WebSocket
         ▼                           ▼
┌──────────────────────────────────────────────────────┐
│                  MATCHING SERVICE                     │
│                                                       │
│  1. User A joins queue                                │
│  2. User B joins queue → MATCH FOUND!                 │
│  3. HTTP POST to Collaboration Service ───────┐       │
│  4. Receive collaborationId                   │       │
│  5. Emit "match_success" to both users        │       │
└───────────────────────────────────────────────┼───────┘
                                                │
                                                ▼
                        ┌───────────────────────────────┐
                        │     COLLABORATION SERVICE     │
                        │                               │
                        │  • Validates users            │
                        │  • Selects question           │
                        │  • Creates session            │
                        │  • Returns collaborationId    │
                        └───────────────────────────────┘
                                                │
                                                ▼
                        ┌───────────────────────────────┐
                        │      QUESTION SERVICE         │
                        │                               │
                        │  • Selects random question    │
                        │    based on topic/difficulty  │
                        └───────────────────────────────┘
```

## Flow Step-by-Step

### 1. Match Found (Matching Service)

When two users with matching criteria are found:

```typescript
// In match.ts
if (status === "matched") {
    const matchId = uuidv4();

    // Create collaboration session via HTTP
    const collaborationId = await createCollaborationSession({
        matchId,
        userAId: req.userId,
        userBId: partnerId,
        difficulty: matchedDifficulty,
        language: matchedLanguage,
        topic: req.topic,
    });

    if (!collaborationId) {
        // Failed - return users to queue
        return { matchFound: false, startTime: Date.now() };
    }

    return {
        matchFound: true,
        matchId,
        collaborationId,
        matchedTopic: req.topic,
        matchedDifficulty,
        matchedLanguage,
        userId: req.userId,
        partnerId,
    };
}
```

### 2. HTTP Request to Collaboration Service

**File:** `src/services/collaborationService.ts`

```typescript
export async function createCollaborationSession(request: CreateSessionRequest): Promise<string | null> {
    const response = await fetch(`${collaborationServiceUrl}/v1/api/sessions`, {
        method: "POST",
        headers: {
            "Content-Type": "application/json",
            "x-internal-service-key": internalServiceKey,
        },
        body: JSON.stringify(request),
    });

    if (!response.ok) {
        return null;
    }

    const data = await response.json();
    return data.session.collaborationId;
}
```

### 3. Collaboration Service Creates Session

**Endpoint:** `POST /v1/api/sessions`

```typescript
// Request body
{
    matchId: "uuid-123",
    userAId: "user_abc",
    userBId: "user_xyz",
    topic: "Algorithms",
    difficulty: "Easy",
    language: "JavaScript"
}

// Response
{
    session: {
        collaborationId: "collab-456",
        matchId: "uuid-123",
        userAId: "user_abc",
        userBId: "user_xyz",
        questionId: "question-789",
        topic: "Algorithms",
        difficulty: "Easy",
        language: "JavaScript",
        status: "active",
        createdAt: "2026-03-21T12:30:00Z"
    }
}
```

### 4. Users Redirected to Collaboration Page

Both users receive the `match_success` event via WebSocket and are redirected:

```typescript
// Frontend - useMatchingQueue.ts
socketInstance.on(SocketEvents.MATCH_SUCCESS, (data) => {
    if (data.collaborationId && onMatchFound) {
        onMatchFound(data); // Triggers navigation
    }
});

// Frontend - MatchingView.tsx
useMatchingQueue(topic, language, difficulty, (payload) => {
    navigate(collaborationRoute(payload.collaborationId));
    // Redirects to /collaboration/{collaborationId}
});
```

## Configuration

### Environment Variables

**Matching Service (`.env`):**
```env
MS_COLLABORATION_SERVICE_URL=http://collaboration-service:3003
MS_INTERNAL_SERVICE_API_KEY=your-internal-service-key
```

**Collaboration Service (`.env`):**
```env
CS_INTERNAL_SERVICE_API_KEY=your-internal-service-key
CS_API_GATEWAY_URL=http://gateway:8080
CS_QUESTION_SELECTION_PATH=/v1/api/questions/internal/select
```

### Docker Compose

```yaml
matching-service:
  environment:
    - MS_COLLABORATION_SERVICE_URL=http://collaboration-service:3003
    - MS_INTERNAL_SERVICE_API_KEY=your-key
  depends_on:
    - collaboration-service

collaboration-service:
  environment:
    - CS_INTERNAL_SERVICE_API_KEY=your-key
    - CS_API_GATEWAY_URL=http://gateway:8080
```

## Files

### Matching Service
- `src/services/collaborationService.ts` - HTTP client for collaboration service
- `src/match/match.ts` - Match logic that calls collaboration service
- `src/managers/socketManager.ts` - Emits match_success to users
- `src/types/match.ts` - Includes collaborationId in MatchResultSuccess

### Collaboration Service
- `src/routes/sessionRoutes.ts` - POST /v1/api/sessions endpoint
- `src/services/collaborationSessionService.ts` - Session creation logic
- `src/services/questionSelectionService.ts` - Calls question service

### Frontend
- `src/services/matching/useMatchingQueue.ts` - Handles match_success event
- `src/views/matching/MatchingView.tsx` - Navigates to collaboration page

## Error Handling

If session creation fails:
1. Matching service returns `matchFound: false`
2. Users stay in queue and can retry
3. Error is logged for debugging

```typescript
if (!collaborationId) {
    socketLogger.error("Failed to create collaboration session");
    return { matchFound: false, startTime: Date.now() };
}
```
